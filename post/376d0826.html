回溯

简介：

​ 回溯，计算机算法，回溯法也称试探法，它的基本思想是：从问题的某一种状态（初始状态）出发，搜索从这种状态出发所能达到的所有“状态”，当一条路走到“尽头”的时候（不能再前进），再后退一步或若干步，从另一种可能“状态”出发，继续搜索，直到所有的“路径”（状态）都试探过。这种不断“前进”、不断“回溯”寻找解的方法，就称作“回溯法”。

​ ——百度百科

​ 回溯与二分查找一样，并不是指某一个单一的算法，而是一种计算机思维，是一种思想。

​ 所以，回溯法在不同的应用场景下，其代码实现很不相同，比二分查找还要复杂。不过万变不离其宗，只要深刻理解了回溯思想，就可以做到以不变应万变，而且回溯法必须要用到递归，所以代码往往很简洁优雅，不过初学难以理解。

在讲解回溯之前，你需要先知道树、二叉树、树的DFS几个概念。

树与二叉树：

​ 树，是一种计算机数据结构，一棵树只能有一个根结点，没有任何分支（没有一个孩子）的结点叫做叶子结点。二叉树是树的一种特殊形式，二叉树：一个结点的孩子结点最多只能有两个，而且有左右之分，允许出现没有左子结点但是有右子结点的情况。

树的DFS:

​ DFS：深度优先遍历（Depth-First-Search，简称 DFS），树的DFS就是对一颗树按照深度优先的方式遍历它的每个结点。

​ 所谓遍历：拿到一棵树，访问它的每一个结点，每一个结点有且只能访问一次，而且不能漏掉任何一个结点。

​ 所谓深度优先：尽可能纵向往深处的去访问树，遇到分支时不用着急访问所有分支，默认选第一个分支先走下去，一直走到死路了，再回过头来走没走过的分支。

在遍历之前：

​ 在遍历树之前，一定要先弄明白的一点是：树的存储方式，如果你没有弄懂树的存储方式，你可能很难理解DFS的伪代码。

​ 一颗树的存储方式有：双亲表示法、子结点表示法、孩子-兄弟表示法，这里我们不过多介绍这些。

​ 总之，一颗树状的数据结构，计算机要存储它，不可能只存每个结点的信息吧，必须要想办法去描述结点与结点的关系，而一颗随机的普通树，结点错综复杂，不可能把每个结点之间的关系都做存储，所以就只能存：能够描述这颗树所必不可少的结点之间的关系。

​ 一般以 “父子关系” 的方式存储，就像这样：先存全部的结点，而每个结点又会有一个列表，里面放它的所有子结点。这样存储，以后随时取出数据就可以直接描绘出一颗完整的树。

​ 也就是说，我们访问一个结点，就一定能拿到它的全部子节点（可以是一个列表记录，也可以是其他方式，反正必须提供给给遍历者，没有子结点的情况就提供 null），只有这样才能继续访问下去。

从树的DFS到回溯法：

​ 其实，回溯 和 回溯法 是两个概念。回溯体现在很多地方，只要是有一个退回到原来的数据的过程，都可以称之为回溯。像KMP算法就有回溯过程，DFS也当然：回溯到之前访问过的结点，去走没走过的分支。

​ 而不是随便一个有回溯过程的算法就可以称为回溯法的，严格意义上的回溯法要包含：递归、回溯、剪枝等过程。

​ 而上面说的树的DFS就包含递归 + 回溯，所以它离我们的回溯法很接近了，可以说回溯法就是在树的DFS基础上的。

​ 我们通过一些例题来讲解回溯法：

例一：

​ 求a、b、c三个个字母的可重复全排列，即从字母中每次选一个字母出来，共选三次，允许出现重复的字母，打印出所有的情况。

​ 上述题目，三个字母的可重复全排列的种数很好求，就是 3 * 3 * 3 = 27 种，但是要把每种组合情况打印出来就必须要用到回溯法了。

​ 我们假设第一个字母选a，第二个字母就有三种选择，以此类推，我们可以将所有的组合情况抽象成三棵树来表示：

​

​ 如上图，我们得到了全部的27种组合。每棵树从根结点开始走到叶子结点结束，这一过程就对应一种组合！要想全部打印出来，就要走完三棵树的所有路径。

​ 注意，之前讲的树的DFS只是访问树的全部结点，而我们现在要考虑的是：路径！这一点非常关键！

重点①：

​ 也就是说，我们要用一个变量时刻保持已经走过的结点，当走到叶子结点时，就意味着我们找到了一种答案，就返回这个变量作为一种答案。

​ 如果你深刻理解了树的DFS的回溯过程，你应该知道：我们不可能每次走完一种答案路径，就新建一个变量来存储这个答案。

​ 正确的做法是全程只用一个变量 path 存储，这个 path 不断修改，当它达成一种组合答案时，就将它放入一个总答案里面，最后我们只需要返回这个由27种答案组成的总答案就可以了。这一点是理解回溯法的重点！为什么只能使用一个变量？

​

​ 如上图，我们的DFS走到死路了，要走之前没有走过的分支，只能先向上回溯一层！不可能直接回溯到根结点！所以每当我们到达叶子结点，发现一种答案，我们要去寻找下一个答案，只能将这个变量的最后一个字母给删去，即让我们的 path 也跟着回溯！模拟DFS的回溯过程。如果是新建一个变量去存储下一种答案的话，那么之前走过的结点怎么办呢？我们又不能直接回溯到根结点，去探索第二条路径，而且你怎么知道刚才的路径被走过了呢？

​ 所以，我们只能在当前的位置上向上做一层的回溯！所以，全程只能用一个path 去跟着整个递归和回溯过程 。并且要在一定的时刻将它放入一个总答案里面。

重点②：

​ 三棵树都是我们模拟的，真实情况是不存在的！也就是计算机没有存储这样三棵树的！那么我们按照DFS去遍历，当访问一个结点时，是无法拿到它的全部子结点的！这我们还怎么遍历呢？以前的 for（node 的所有子结点）肯定不能用了，那 for 循环怎么写呢？这是回溯法的第二个重点！每种题目，for循环不同！

​ 我们的 for 语句必须要满足每一次递归调用，也就是说，我们必须时刻知道每一个结点的子结点情况。这道题还比较容易，因为每个结点的都固定拥有三个子结点，而且我们知道它们是 a、b、c三个字母！如果题目以数组的形式给出a、b、c三个字母，那么这个数组就代表每一个结点的子结点情况。那么我们for 循环就可以完美地写成 for（ 这个数组 ）。

​ 上面两题，可以看到一种容易的情况：那就是所有结点的子结点数量一样，即分支数一样。一种稍微复杂一点需要剪枝的情况：子结点数量在变化，但是始终在一个给定的数组做不断减少的变化。

​ 所以上述两例只是回溯法的开门见山，还有更为复杂的情况：

​ 如果每一个结点的子结点数在变化，减少的数量都不是均匀的，更或许不仅仅只减少，或许有的子结点是a、b、c，而有的子结点却是其他的一些字母！这时候我们的 for 循环怎么去适应每一次递归调用呢？

​ 这里我们不深入探究，因为每一题给的条件都不同，但是要知道，要是题目中每个结点的子结点都不同，且没有规律变化性，那么题目一定会以一种形式给出所有的子结点的！就像我们遍历树，一定要有一棵真实存储的树一样。

​ 说一种处理方法：首先是关于剪枝问题，如果一直在给定一堆的字母中选，就需要考虑字母重复问题，此时有可能就要用到剪枝，而如果题目要求在一堆字母中选一个，又在另外完全不同的一堆字母中选，像这种完全没有规律的变化，那就不用考虑剪枝问题了，我们要考虑怎么让 for 循环每次都拿到指定的那一堆字母！可以通过回溯法函数的参数传入这么一堆字母（或者是其他形式），那么我们在递归调用时，一定要记住这个参数肯定要不断变化！而不是像上面的例题中，每次递归都是 backTrack（nums[], path, res）; 我们的回溯函数有可能会定义成 backTrack（nums[], path, res ，x）{ ……} ，x 表示一个新的参数，而递归调用中，x 就要变化！可能自增也可能自减或者是其他变化，来满足题目。

总结：

​ 回溯法，其实就是模拟树的DFS过程！

​ 树的DFS中，循环 + 递归 就可以实现：递归往下探索 和 回溯上一层 这么一整套遍历过程！

​ 回溯法中，牢记：树是虚拟的，结点也是不存在的，我们真正要的是 path！是一条一条的路径！那么 path 在形成过程中就要时刻跟随和模拟这个遍历过程！遍历在递归往下，path 就要做一定的操作！遍历在回溯往上，path 就要做与之前完全相反的操作，来模拟回溯。

​ 可以记住一种模板：

​ 这里再说一个非常细节的点，很多初学者都很疑惑的点：

​ 在我们的 java 中，很多时候我们的 path 都用一个 ArrayList 来做路径存储，而总答案 res 当然是一个两重嵌套的 ArrayList，如下定义：

​ 定义当然是没有问题，但是我们在回溯算法中当 path 满足一种答案，要将其放入总答案时，要注意！一定不能写成：res.add(path);

​ 因为，path 放入res 中后，path 还要变化的，还要去构成另一种答案的！而此时path 一旦改变，刚才放入 res 中的那个 path 就被改变了！

​ 所以，正确的写法是：res.add(new ArrayList(path)); 这样，每次加入 res 的都是一个和 path 数据一样但是是新建的 ArrayList 集合。
